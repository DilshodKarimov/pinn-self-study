##  Описание выбранного СДУ (процесс Орнштейна-Уленбека)

Процесс Орнштейна-Уленбека – стационарный Гаусс-Марковский процесс, который описывает случайное блужание с возвратом к среднему значению. В отличие от обычного броуновского движения, он имеет «силу притяжения», которая стремится вернуть траекторию системы к ее долгосрочному среднему значению.

Процесс задается с помощью стохастического дифференциального уравнения:

![alt text](https://github.com/DilshodKarimov/pinn-self-study/tree/main/images/image.png)

Где:
* $X(t)$ – текущее состояние процесса.
* $\theta$ – скорость возврата к среднему.
* $\mu$ – долгосрочное среднее значение.
* $\sigma$ – волатильность (величина случайных возмущений).
* $W(t)$ – Винеровский процесс (броуновское движение).

---

##  Описание генератора данных

Для того чтобы смоделировать процесс, необходимо дифференцировать уравнение (1) по времени. Для этого используется метод Эйлера-Маруямы. Таким образом, уравнение Орнштейна-Уленбека принимает дискретизированный вид:

![alt text](https://github.com/DilshodKarimov/pinn-self-study/tree/main/images/image-1.png)

Функция генератора `generator()` использует в качестве аргументов заданные вручную значения для $\theta$, $\mu$ и $\sigma$. Также задаются:
* $X(0)$ – начальное приближение.
* $T$ – общее время.
* $dt$ – шаг по времени.
* `filename` – название файла формата `.csv`, куда построчно записываются данные по столбцам `t` (момент времени) и `X(t)` (значение процесса). Количество строк составляет $N = T/dt$.

Для математических операций используется библиотека `numpy`, а для работы с таблицами — `pandas`. Также генератор имеет функционал наглядной демонстрации сгенерированных данных при помощи графика с использованием библиотеки `matplotlib`.

##  Описание модели нейросети

Для решения задачи нахождения параметров СДУ была использована модель PINN (Physics-Informed Neural Network) с архитектурой, состоящей из последовательных полносвязных слоев:
* **Входной слой:** принимает 1 значение – момент времени $t$.
* **Скрытые слои:** 2 полносвязных слоя по 64 нейрона, выполняющие операцию гиперболического тангенса (`Tanh`).
* **Выходной слой:** возвращает 1 значение – состояние процесса $X(t)$.

Также модель имеет 3 обучаемых параметра: `theta_pred`, `mu_pred` и `sigma_pred` ($\theta$, $\mu$ и $\sigma$, предсказанные моделью). Они являются скалярами (единичными тензорами) с включенным флагом `requires_grad=True` для того, чтобы `Autograd` мог отслеживать операции с ними при обратном проходе во время оптимизации. Работа модели осуществляется посредством библиотеки `torch`.

##  Чтение файла и построение сетки данных

Чтение данных из файла `.csv` выполняется с помощью функции `pandas.read_csv()`. При успешном прочтении программа уведомляет об этом и продолжает работу. Если возникает ошибка, связанная с отсутствием файла, программа перехватывает её, предупреждает пользователя и завершает работу.

Сразу после успешного чтения столбцы таблицы `t` и `X(t)` преобразуются в соответствующие тензоры размером в $N$ строк и 1 столбец (векторы). Следом идет создание сетки точек для обучения.

##  Функция loss, оптимизатор и обучение

После инициализации модели и чтения файла задается функция `total_loss()`, вычисляющая общие потери:

![alt text](https://github.com/DilshodKarimov/pinn-self-study/tree/main/images/image-3.png)

* **$Loss_{data}$:** потери по данным — отклонение предсказанных значений от истинных экспериментальных, вычисляемые через среднеквадратичную ошибку MSE.
* **$Loss_{physics}$:** потери по физике процесса. 

Поскольку само СДУ стохастическое и содержит недифференцируемый винеровский процесс $W(t)$, прямое дифференцирование через функцию `autograd()` приводило к появлению значений `NaN` или крайне низкой сходимости. В связи с этим было принято решение использовать уравнение Фоккера-Планка в частных производных:

![alt text](https://github.com/DilshodKarimov/pinn-self-study/tree/main/images/image-7.png)

В качестве оптимизатора выбран алгоритм **Adam** ввиду наличия учета импульса и адаптивной скорости обучения. В теле оптимизатора заложено запоминание значений параметров СДУ и общих потерь для демонстрации хода обучения. Также присутствует цикл отладки, выводящий значения $Loss$ и ошибки предсказанных параметров относительно истинных. По окончании обучения выводятся искомые значения параметров и итоговый лосс.
